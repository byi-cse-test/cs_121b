Controlling Your Code

Choice Making and Computational Flow

C and Python share a set of structures used to control which part of your code is executed, based on specific situations. The idea of situational 'computational flow' control isn't unique to Python or C. It is found in all languages though the way it is accomplished can be different.

Logic Used in Computing

Boolean logic is used in most languages. In boolean logic, the result of a claim is either true or false, there are no maybe's or kinda's. Something is either true or it is false. This bipartate truth concept isn't the only one, databases use a tripartate concept of truth, but its simplicity is embraced in most programming languages.

C, like Python, uses a set of operators to check for truth...

Getting Loopy

Unlike Python, C has no list. Therefore there are no filter or reduce functions or methods. You can find these in non- standard libraries, or you could write your own versions of filter and reduce. In C, we use raw loops, not map, filter, or reduce.

While

C's while loop works with a single boolean logic check. As long as the logic check result is true, the loop continues looping.

For

Since C has not lists, it also has no version of Python's for-in loop. Instead, C has a for loop you use to accomplish the sam thing. C's for loop has multiple components.
